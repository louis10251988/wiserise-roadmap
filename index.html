👥 People 人事部
Hiring → Onboarding → Scheduling → Discipline → Exit · All 5 stores
80/20 关键：80%的问题来自"缺人"和"招错人"。把招人预算和入职流程管好，其他事少一半。
80/20 Core: 80% of problems come from being understaffed or hiring the wrong people. Fix the hiring budget and onboarding process, and half your problems disappear.
主要工作流程 · Main Workflow (in order)
  1. Hiring 招人 — GM says "need 2 server" in store channel → Hermes checks budget → over budget = stop → OK = create job post
  2. Screening 筛人 — Hermes scores & ranks candidates → GM picks for interview → GM says "hire" in Slack → writes to Brain
  3. Onboarding 入职 — Hermes generates I-9 / 7shifts account / PIN → GM helps new hire fill forms → photo to Slack → Hermes records Brain
  4. Scheduling 排班 — GM schedules in 7shifts (own store) → system sends to staff → Hermes checks staffing → short = 7AM alert to you
  5. Daily Mgmt 日常管理 — GM manages attendance/discipline → minor issues = coach (teach first) → repeat = write-up (paper + photo to Slack)
  6. Exit 离职 — floor staff exit → Coach Sally approves → manager exit → Louis approves → Hermes generates exit docs
规章制度 · Rules
Rule 1 / 规矩1：Check budget before hiring. Over budget = stop, no discussion. / 招人先查预算。超预算 = 停。
Rule 2 / 规矩2：Onboarding must include I-9 + 7shifts account + PIN. Missing any = not onboarded. / 入职必须填I-9 + 7shifts + PIN。
Rule 3 / 规矩3：Discipline: teach first, punish second. 1st = coach (verbal), 2nd = write-up (paper + photo), serious = direct write-up. / 纪律先教后罚。
Rule 4 / 规矩4：Fire floor staff → Coach Sally can decide. Fire manager → Louis decides. Never auto-fire. / 开除floor→Coach，开除manager→Louis。
Rule 5 / 规矩5：GM sees own store only. Other stores = denied. / GM只看自己店。

  4. Payroll 工资 — Gusto runs payroll → Hermes checks timesheets → anomalies → you → normal = silent
  5. Tax 税务 — Tax docs → Yumi handles → involves bank/IRS → Louis approves
  6. Monthly Report 月报 — P&L → Hermes pulls from Brain → sends to your DM
规章制度 · Rules
Rule 1: Bank/wire/ACH → Louis + Luna only. No one else. / 银行→Louis+Luna only
Rule 2: Purchase ≥$500 → Louis. <$500 → Yumi. No exceptions. / 采购≥$500→Louis
Rule 3: Tax/W-2/lease/contract → sending externally needs Louis first. Yumi can use internally. / 税务发外面→先过Louis
Rule 4: Bank statements / full balance → Yumi cannot forward to personal WeChat/email. / 银行流水Yumi不能转发
Rule 5: OpenBudget → Louis + Luna only. Others = denied. / OpenBudget→Louis+Luna

📦 Procurement 采购部

80/20 关键：80%的采购浪费来自"没说清楚就买了"。把需求→审批→下单的链条管死，浪费就少。
80/20 Core: 80% of procurement waste comes from unclear requests. Lock down the request→approve→order chain.
主要工作流程 · Main Workflow
  1. Request 提需求 — Store says "买一个冰箱" / "need 50lb flour" in Slack → Hermes auto-creates purchase order
  2. Review Yumi审 — Yumi locks tax and total → checks for cheaper options → confirms amount

  4. Order 下单 — Hermes records → notifies store "ordered" → arrives → store confirms
  5. Pay 付款 — Pay with invoice → record in Brain → reconcile
  6. Notify 通知 — Every step notifies the requester (status, ETA)
规章制度 · Rules
Rule 1: No purchase order = no buy. Create order first. / 没采购单不能买
Rule 2: ≥$500 → Louis. No exceptions. <$500 → Yumi. / ≥$500→Louis
Rule 3: Every purchase needs invoice/receipt. No proof = no pay. / 凭证才能付
Rule 4: Yumi locks tax and total. Cannot change amount randomly. / Yumi锁税+总额
Rule 5: Delivery must be confirmed. Not arrived = chase. / 到货要确认

🖥️ IT 技术部

80/20 Core: 80% of system issues come from wrong permissions and expired keys. Manage these two and the system is stable.
主要工作流程 · Main Workflow
  1. Hermes — The only worker. Skills = how to do work. Cron = when to do work. No other agents. / 唯一worker

10 条规矩 · 10 Work Rules (auto-checked nightly)
  1. Use Hermes only, not GLM web chat / 只用Hermes
  2. Check data before building / 先查数据
  3. Never rebuild from scratch, patch what's broken / 不推倒重来
  4. One session = one task, close when done / 一session一事
  5. Roadmap before building / 先有roadmap
  6. Desktop = build, Slack = operate, don't mix / Desktop建Slack管
  7. Verify, don't assume / 查数据别猜
  8. Fix one thing at a time, test, then next / 一次改一个
  9. Brain is truth, docs lie / Brain是真相
  10. Done means move on / 做完就走
规章制度 · Rules
Rule 1: Change settings/code → Louis only. Luna has all other rights except code. / 改代码→Louis only
Rule 2: Permission changes in 4 places: owner-gate → openbudget-guard → config.yaml → .env. Miss one = half-broken. / 改4个地方
Rule 3: After permission change, restart gateway from OUTSIDE Slack (restart inside = kills itself). / 外面重启
Rule 4: Managers in store channel + company workspace only. Never in #02-executive. / 经理不进#02-exec
Rule 5: New keys → save to 1Password first → then sync to .env. Never edit .env directly. / 先存1Password
